Sundara Vira
Gameplay Design Document (Browser-First)

## 1. Vision
Sundara Vira is a browser action adventure inspired by Hanuman's journey in Sundarakanda.
The core fantasy is simple: devotion powers movement, combat, and transformation.

Design pillars:
- Devotion as gameplay: prayer input is a meaningful mechanic, not only narrative flavor.
- Heroic mobility: leap, glide, wall-run, and size-shift make traversal central.
- Dharma over destruction: player choices shape tone, rewards, and endings.

## 2. Audience and Platform
Primary audience:
- Ages 12+ who enjoy mythic action, accessible controls, and short sessions.

Target platform:
- Any modern browser (desktop and mobile), no install required.
- PWA support for near-app experience.

Performance targets:
- 60 FPS on mid-range desktop.
- 30-60 FPS on mid-range mobile.
- Initial load under 8 seconds on standard 4G.

## 3. Browser-Friendly Tech Stack (Tool Choices)
Recommended stack for easy deployment and cross-device play:

Core game tech:
- Engine: Phaser 3 (2D WebGL/canvas fallback, mature for browser action games).
- Language: TypeScript.
- Build tool: Vite.
- UI layer: Phaser UI plus lightweight DOM overlays.

Input and audio tools:
- Speech recognition: Web Speech API for chant detection (with typed fallback always available).
- Audio: Howler.js for music, stingers, and spatial effects.
- Haptics (mobile): Vibration API for impact moments.

Content and data:
- Save system: localStorage for profile/checkpoint, optional cloud sync later.
- Config/content: JSON-driven level scripts, enemy tables, and verse prompts.

Distribution:
- Hosting: GitHub Pages, Netlify, or Vercel (static deployment).
- PWA: service worker for caching and fast repeat launch.

Dev workflow:
- Repo: GitHub.
- CI: GitHub Actions for build and deploy.
- Testing: Vitest (logic), Playwright (smoke test in browser).

## 4. Core Game Loop
1. Enter zone and receive objective.
2. Traverse obstacles using movement and size-shift.
3. Build Bhakti Energy through accurate chant/typed input.
4. Use powers in combat, puzzle gates, or escapes.
5. Make Dharma-impacting choices.
6. Finish encounter, earn upgrades, unlock next chapter.

Session length target:
- 10-15 minutes per mission.
- 60-90 minutes for complete chapter arc.

## 5. Controls and Accessibility
Desktop controls:
- Move: WASD/arrow keys.
- Jump/leap: Space.
- Light/heavy attack: J/K.
- Size up/down: Q/E.
- Chant input: hold R to speak or press T to type.

Mobile controls:
- Virtual stick for movement.
- Three main action buttons: jump, attack, power.
- Swipe up/down for quick size change.
- Tap mic button for chant; keyboard input as fallback.

Accessibility:
- Full typed-input mode if microphone is unavailable.
- Adjustable chant recognition sensitivity.
- Subtitle and transliteration options.
- Color-blind friendly UI palette and scalable text.

## 6. Player Systems

### 6.1 Bhakti Energy
Generated by:
- Successful chant recognition (voice).
- Correct verse typing.
- Perfect dodge/combo moments.

Spend on:
- Healing pulse.
- Shield aura.
- Anjaneya State (temporary empowered mode).

Balance rule:
- Voice gives faster gain but higher risk of misread.
- Typed gives slower, consistent gain.

### 6.2 Size Transformation
Large form:
- Crowd control, heavy damage, breakable walls.

Small form:
- Stealth routes, narrow tunnels, evasion bonus.

Mastery challenge:
- Several encounters require rapid switching between forms.

### 6.3 Dharma Alignment
Increases with:
- Protecting innocents.
- Showing restraint.
- Speaking truth in key story moments.

Reduces with:
- Excess collateral damage.
- Optional cruel choices.

Effects:
- Branching dialogues.
- Different reward blessings.
- Different ending tone.

## 7. Chapter Design (Narrative + Play)

### Prologue: The Forgotten Power
Story:
- Jambavan awakens Hanuman's confidence at the seashore.

Gameplay goals:
- Teach movement, leap charge, basic combat, and first chant.
- End with ocean mega-jump.

### Chapter 1: Trial of the Ocean
Story:
- Hanuman faces Mainaka, Surasa, and Simhika.

Gameplay goals:
- Introduce situational choice, size-shift puzzle, and aerial danger combat.

### Chapter 2: The City of Illusions
Story:
- Hanuman infiltrates Lanka and defeats Lankini.

Gameplay goals:
- Stealth routes, patrol timing, mantra puzzle gates.

### Chapter 3: Ashoka Vatika
Story:
- Hanuman finds Sita and earns her trust.

Gameplay goals:
- Low-combat narrative sequence, precision verse prompts, trust meter.

### Chapter 4: The Roar in Lanka
Story:
- Hanuman reveals power and warns Ravana.

Gameplay goals:
- Arena combat, elite enemy variants, capture and court scene.

### Chapter 5: The Fire of Dharma
Story:
- Tail ignited; Lanka escape and controlled destruction.

Gameplay goals:
- High-speed parkour with risk/reward pathing and civilian protection scoring.

### Final Chapter: Return Across the Sea
Story:
- Hanuman returns with Sita's message.

Gameplay goals:
- Endurance flight, pursuit enemies, finale cinematic unlock.

## 8. Enemy and Encounter Design
Enemy classes:
- Guard: basic melee, tutorial combat.
- Archer: ranged pressure, anti-air threat.
- Mystic: debuff and illusion traps.
- Elite Rakshasa: form-specific resistances.
- Bosses: Surasa, Simhika, Lankini, Aksha Kumara.

Encounter pattern:
- Traverse -> discover -> solve/fight -> moral choice -> reward.

## 9. Progression and Rewards
Progression tracks:
- Power track: unlock abilities (shield aura, leap burst, blaze sprint).
- Skill track: combo depth, reduced cooldowns, form mastery.
- Dharma track: blessings, alternate scenes, ending tier.

Collectibles:
- Sundara Fragments: lore pieces and challenge keys.
- Vanara Tokens: upgrade currency.

## 10. Endings
1. Sacred Victory: high Dharma and protected civilians, true ending cinematic.
2. Warrior Path: mission success with moderate Dharma, combat-forward close.
3. Flame Without Balance: success with heavy collateral, somber ending.

## 11. MVP Scope (Browser Launch)
MVP content:
- Prologue + Chapter 1 + one Lanka mission slice.
- 3 playable forms/states: normal, large, small.
- 2 boss fights.
- Voice and typed Bhakti input.
- One ending path for first release.

MVP success metrics:
- D1 retention above 25%.
- Average session above 9 minutes.
- Input completion success above 80% for typed mode.

## 12. Implementation Plan (Practical)
Phase 1 (2-3 weeks):
- Phaser project setup, controls, camera, and movement prototype.

Phase 2 (2-3 weeks):
- Bhakti system with Web Speech API + typed fallback.

Phase 3 (3-4 weeks):
- Chapter content scripting, enemy AI, and boss pattern system.

Phase 4 (2 weeks):
- UI polish, localization hooks, balancing, and performance pass.

Phase 5 (1 week):
- PWA setup, CI/CD deploy, browser compatibility QA.

## 13. Risks and Mitigations
Risk: speech recognition inconsistency across browsers.
Mitigation: typed mode always available; optional language-specific phrase packs.

Risk: mobile performance drops in heavy combat.
Mitigation: sprite atlases, pooled particles, adaptive effects quality.

Risk: sensitive cultural representation.
Mitigation: involve domain reviewers for verses, visuals, and narrative tone.
